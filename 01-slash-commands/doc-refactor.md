# 文档重构

根据项目类型重构项目文档结构：

1. **分析项目**：识别类型（库/API/Web 应用/CLI/微服务）、架构和用户画像
2. **集中文档**：将技术文档移至 `docs/`，并提供正确的交叉引用
3. **根目录 README.md**：精简作为入口点，包含概述、快速入门、模块/组件摘要、许可证、联系人
4. **组件文档**：为每个模块/包/服务添加 README 文件，包含设置和测试说明
5. **组织 `docs/`**：按相关类别分类：
   - 架构、API 参考、数据库、设计、故障排除、部署、贡献指南（根据项目需要调整）
6. **创建指南**（选择适用的）：
   - 用户指南：面向最终用户的应用程序文档
   - API 文档：端点、认证、API 示例
   - 开发指南：设置、测试、贡献工作流程
   - 部署指南：服务/应用程序的生产部署
7. **使用 Mermaid** 绘制所有图表（架构、流程、模式）

保持文档简洁、可扫描，且与项目类型相关。
